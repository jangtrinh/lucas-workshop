Content Creation Arsenal

Level 3 · From zero assets to a complete content library

LEVEL 3 — INTRO
TRANSCRIPT

Chào mừng đến với Level 3. Content Creation Arsenal. Ở level này, bạn sẽ xây dựng pipeline tạo content hoàn chỉnh: từ một file Markdown duy nhất, bạn sẽ tạo ra audio, hình ảnh, và video. Bạn sẽ xây knowledge base cho sản phẩm của mình, và thiết lập pipeline từ research đến launch campaign trong một tuần. Mục tiêu là không còn tạo content thủ công nữa.

LEARNING OBJECTIVES
  1. Build an asset pipeline that generates audio, images, and video from a single Markdown source
  2. Create a product knowledge base that AI can query with citations
  3. Execute a research-to-launch campaign pipeline using AI tools

3.1 THE ASSET PIPELINE

SESSION 3.1
TRANSCRIPT

Session 3.1. The Asset Pipeline. Multimedia từ một file Markdown duy nhất.

NGHE — ATTENTION HOOK
TRANSCRIPT

Một founder solo cần ba tuần để tạo content cho một product launch: bài blog, podcast, ảnh mạng xã hội, video demo. Sau khi xây asset pipeline, công việc đó chỉ còn mất bốn tiếng. Không phải vì AI là phép màu. Mà vì pipeline loại bỏ các công việc lặp đi lặp lại.

ATTENTION HOOK — 3 WEEKS TO 4 HOURS

A solo founder needed 3 weeks to create assets for a product launch: blog post, podcast, social images, demo video. After building an asset pipeline, the same work took 4 hours.

Not because AI is magic — because the pipeline eliminates redundant work. You write once. The pipeline produces everything.

Why This Matters

Lucas's pain point: teaching materials look amateurish. Professional content requires audio, visuals, and video. Creating each separately is 10x slower than building a pipeline — because every piece of content needs to align with every other piece, and that alignment work is what kills time.

Core Concept: The Asset Pipeline

NGHE — KHÁI NIỆM CHÍNH
TRANSCRIPT

Khái niệm chính: Asset Pipeline. Bạn chỉ viết một file Markdown duy nhất. File đó là source of truth. Từ đó, bạn trích xuất headings và key points, rồi chuyển text thành audio qua TTS, tạo hình ảnh qua Imagen hoặc Nano Banana, tạo video ngắn qua Veo, và tạo nhạc nền qua Lyria. Kết quả là toàn bộ content cho một launch đến từ một nguồn duy nhất, đảm bảo tính nhất quán.

SOURCE (Markdown/content)
    ↓
┌───────────────────────────────┐
│ Layer 1: TEXT → Structured    │  Extract headings, key points, scripts
│ Layer 2: TEXT → AUDIO (TTS)   │  VieNeu-TTS, Gemini TTS, NotebookLM
│ Layer 3: TEXT → IMAGES        │  Nano Banana, Imagen 4 (diagrams, infographics)
│ Layer 4: TEXT → VIDEO         │  Veo 3.1 Lite (5-8s explainer clips)
│ Layer 5: TEXT → MUSIC         │  Lyria 3 (background audio, intros)
└───────────────────────────────┘
    ↓
DISTRIBUTION (website, social, email, classroom)

TRANSCRIPT

Ý tưởng đến từ ngành điện ảnh. Các xưởng phim không quay cảnh ngẫu nhiên. Họ chia kịch bản thành shot list, rồi sản xuất song song: âm thanh, hình ảnh, nhạc, hiệu ứng hình ảnh. Pipeline content của bạn hoạt động giống hệt vậy. File Markdown của bạn chính là kịch bản.

CROSS-DOMAIN — FILM PRODUCTION

Film studios don't shoot scenes randomly. They break a script into a shot list, then produce assets in parallel: sound, visuals, music, VFX. Your content pipeline works the same way — the Markdown file IS your script. Structure first, then parallel production.

BEST PRACTICE
DO THIS

Start with ONE source file per product or topic. All assets derive from it. When your source changes, every asset updates from the same truth.

AVOID THIS

Creating assets in isolation — separate blog post, separate podcast script, separate video script. That guarantees inconsistency. The podcast will contradict the blog. The video will miss the key point. One source prevents all of this.

TRANSCRIPT

Chọn một dự án của bạn. Viết một file Markdown hai trăm từ với: tiêu đề, ba điểm chính, một câu trích dẫn, và một call-to-action. Sau đó mô tả ba assets bạn sẽ tạo từ file đó: audio narration, social image, và explainer video. Viết prompt cụ thể cho từng asset, không phải chung chung.

3.2 KNOWLEDGE BASE FOR YOUR PRODUCT

SESSION 3.2
TRANSCRIPT

Session 3.2. Knowledge Base cho sản phẩm của bạn. Từ những cuộc trò chuyện tạm thời đến một product brain có thể truy vấn được.

NGHE — ATTENTION HOOK
TRANSCRIPT

Một PM tại một SaaS startup có hơn hai trăm support ticket, năm mươi trang Notion, và ba mươi Slack thread về sản phẩm. Khi investor hỏi tỷ lệ retention theo cohort, không ai trả lời được mà không mất hai tiếng để đào bới. Giải pháp: xây knowledge base có cấu trúc, AI truy vấn trong ba giây, kèm citation chỉ rõ nguồn tài liệu.

ATTENTION HOOK — THE UNANSWERABLE INVESTOR QUESTION

A PM at a SaaS startup had 200+ support tickets, 50 Notion pages, and 30 Slack threads about the product. When investors asked "What's your retention rate by cohort?", nobody could answer without 2 hours of digging.

The fix: a structured knowledge base that AI could query in 3 seconds, with citations pointing to the exact source document.

Why This Matters

Lucas's pain: generic AI outputs for Elfie products. AI gives generic answers because it has no product-specific context. In Session 1.1, you built a CONTEXT.md. Now we scale that to a full, queryable knowledge base — one that any stakeholder can interrogate with AI and get accurate, cited answers.

Core Concept: The Product Brain

NGHE — KHÁI NIỆM CHÍNH
TRANSCRIPT

Khái niệm chính là Product Brain gồm ba lớp. Sources là nơi bạn tập hợp tài liệu: docs, CSV, PDF, tickets, research. Knowledge Base là nơi Gemini File Search tự động chunk, embed, và index các file đó. Query Layer là nơi bất kỳ ai cũng có thể hỏi và nhận được câu trả lời kèm citation. Gemini File Search cho phép query miễn phí sau khi đã index. Context Caching giảm chín mươi phần trăm chi phí cho các query lặp lại.

NGHE — CROSS-DOMAIN INSIGHT
TRANSCRIPT

Insight đến từ ngành thư viện học. Thư viện không chỉ lưu sách, họ catalog chúng với metadata: chủ đề, tác giả, ngày, mã phân loại. Knowledge base không có cấu trúc chỉ là một thư mục file. Index mới là sản phẩm thực sự. Khi bạn đặt tên file đúng và tổ chức theo thư mục rõ ràng, AI mới có thể tìm đúng nguồn và trích dẫn chính xác.

CROSS-DOMAIN — LIBRARY SCIENCE

Libraries don't just store books — they catalog them with metadata (subject, author, date, call number). A knowledge base without structure is just a folder of files. The index IS the product. Consistent naming and folder organization make the difference between a searchable brain and a pile of documents.

Organize documents by type before uploading: research/, decisions/, metrics/, feedback/. Use consistent naming conventions. Structure makes retrieval accurate.

AVOID THIS

3.3 RESEARCH-TO-LAUNCH CAMPAIGN

SESSION 3.3
TRANSCRIPT

Session 3.3. Research-to-Launch Campaign. Từ nghiên cứu thị trường đến campaign thật trong một pipeline hoàn chỉnh.

NGHE — ATTENTION HOOK
TRANSCRIPT

Hầu hết PM coi research, content creation, và distribution là ba dự án riêng biệt. Đó là lý do campaigns mất sáu tuần. PM giỏi nhất xây pipeline: research nuôi content, content nuôi distribution, distribution tạo ra data nuôi research tiếp theo. Một vòng mất một tuần. Nếu research-to-launch của bạn mất sáu tuần, đối thủ có pipeline ship nhanh gấp sáu lần.

ATTENTION HOOK — THE 6-WEEK TRAP

Most PMs treat research, content creation, and distribution as three separate projects. That's why campaigns take 6 weeks. The best PMs build a pipeline: research feeds content, content feeds distribution, distribution generates data that feeds the next research cycle.

One cycle: 1 week. If your research-to-launch takes 6 weeks, competitors with pipelines ship 6x faster.

Why This Matters

Lucas's pain: can't compete in market. Competition happens at the speed of shipping. A pipeline doesn't just save time — it compounds. Cycle 2 is better than Cycle 1 because Cycle 1's data feeds it. Manual processes don't compound; they plateau.

Core Concept: The Campaign Pipeline

NGHE — KHÁI NIỆM CHÍNH
TRANSCRIPT

Campaign Pipeline gồm bốn giai đoạn trong một tuần. Research: dùng Google Search Grounding, URL Context, và Batch API với chi phí giảm năm mươi phần trăm. Create: dùng Asset Pipeline từ Session 3.1. Launch: deploy lên GitHub Pages từ Level 2. Measure: dùng Structured Output JSON để capture data và feed vào vòng research tiếp theo. Vòng lặp này là vũ khí cạnh tranh thực sự của bạn.

  Launch    → GitHub Pages (from Level 2), social distribution

TRANSCRIPT

Insight từ Lean Startup. Build-Measure-Learn loop của Eric Ries không chỉ áp dụng cho sản phẩm — nó áp dụng cho content cũng vậy. Mỗi piece of content là một hypothesis. Distribution là thí nghiệm. Analytics là bài học. Pipeline làm cho vòng lặp đủ nhanh để iterate từng tuần thay vì từng quý.

CROSS-DOMAIN — LEAN STARTUP

Eric Ries's Build-Measure-Learn loop isn't just for products — it's for content too. Every piece of content is a hypothesis. Distribution is the experiment. Analytics is the learning. The pipeline makes the loop fast enough to iterate weekly.

BEST PRACTICE
DO THIS

Start with ONE campaign type (e.g., "launch blog post + 3 social posts + 1 email"). Perfect that pipeline. Then add campaign types after the first cycle validates your approach.

AVOID THIS

Trying to automate everything at once. A pipeline with 10 untested steps is a debugging nightmare. Start narrow, expand after validation. Complexity added before the first cycle is waste.

TRANSCRIPT

Thiết kế một campaign một tuần cho một trong các dự án của bạn. Xác định mục tiêu với chỉ số cụ thể. Liệt kê ba câu hỏi research để grounding với Google Search. Định nghĩa năm assets từ Asset Pipeline. Viết measurement schema dạng JSON. Cuối cùng, mô tả cụ thể: dữ liệu nào từ Cycle 1 sẽ thay đổi gì trong Cycle 2.

LEVEL 3 SELF-CHECK QUIZ
Q1: What is the main advantage of an Asset Pipeline over creating content one-at-a-time?
  • A) It is faster
  • B) All assets derive from one source, ensuring consistency
  • C) It uses more AI tools
  • D) It costs less per asset
B — Speed is a byproduct. The real advantage is consistency: every asset (audio, image, video) derives from the same Markdown source, so they cannot contradict each other. One-at-a-time creation guarantees drift between assets.
Q2: File Search (RAG) is better than copy-pasting docs into AI because:
  • A) It handles more text per session
  • B) It auto-chunks, embeds, and returns citations pointing to source documents
  • C) It is entirely free
  • D) It works offline
B — File Search indexes documents once (auto-chunking + embedding), then answers queries instantly with citations. Copy-pasting burns full token cost every session and returns no source attribution.
Q3: Context Caching saves money by:
  • A) Using a cheaper underlying model
  • B) Caching previous AI responses
  • C) Reducing input token cost by 90% on repeated large contexts
  • D) Compressing your source documents
C — Context Caching stores a large context (e.g., your 50-document knowledge base) server-side with a 5-minute TTL. Subsequent queries read from cache at ~10% of normal input token cost, not from fresh uploads.
Q4: In the Campaign Pipeline, what feeds the next research cycle?
  • A) More refined AI prompts
  • B) Social media comments
  • C) Structured measurement data (JSON) captured from the previous cycle
  • D) Competitor analysis reports
C — Structured Output (JSON) from Measure phase captures actionable metrics (conversion rates, engagement by asset type, drop-off points). That data becomes the research brief for Cycle 2, making each cycle more targeted than the last.
Q5: You need to create 5 social images. Which tool is most cost-effective?
  • A) Nano Banana at $0.04/image
  • B) Imagen 4 at $0.02/image
  • C) Extracting frames from Veo 3.1 video
  • D) Manual Canva design
B — Imagen 4 at $0.02/image is half the cost of Nano Banana for bulk generation. Use Nano Banana for one-off, highly customized images where control matters more than cost. Veo screenshots waste budget on video generation when a still image is the goal.
