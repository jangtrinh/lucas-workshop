A solo founder spent 10+ hours/week fixing CSS inconsistencies. Every AI-generated page used different colors, spacing, and button styles. The "fix" seemed obvious: be more specific in prompts.

Core Concept: Design Tokens

Design tokens are named variables that replace hardcoded values. Instead of AI guessing "what shade of gray?", it uses your defined --slate: #4E4B66.

/* в•җв•җв•җ DESIGN TOKENS в•җв•җв•җ */
:root {
  /* Colors (use ONLY these) */
  --black: #000000;
  --white: #ffffff;
  --slate: #4e4b66;
  --border: #e0e0e0;
  --bg-muted: #fafafa;

  /* Typography */
  --font-heading: 'Barlow Condensed', sans-serif;
  --font-body: system-ui, -apple-system, sans-serif;
  --font-code: 'SF Mono', 'Fira Code', monospace;

  /* Spacing Scale (8pt grid) */
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 24px;
  --space-xl: 32px;
  --space-2xl: 48px;

  /* Layout */
  --max-width: 800px;
  --sidebar-width: 240px;
}

Key insight: When you tell AI "use --slate for secondary text", it does. When you tell AI "use a grayish color", it invents a different gray every time.

CROSS-DOMAIN вҖ” ARCHITECTURE'S PATTERN LANGUAGE

In 1977, Christopher Alexander published A Pattern Language вҖ” every building element has a NAME, a CONTEXT (when to use it), and a RULE (how). Apply the same: each design component has a class name, a usage context, and styling rules.

Component Vocabulary

Define each component your AI can use. This becomes your Layer 2:

PatternWhenRule
.whyExplaining rationale/contextLeft black border, muted bg, max 3 sentences
.reflectionExercise with user inputContains textarea, has success criteria
.code-blockShowing copyable codeDark bg, copy button, monospace font
.best-practiceKey takeawayBold label, DO THIS / AVOID THIS pair
.insightCross-domain insightMuted bg, left border, max 2 sentences
BEST PRACTICE
DO THIS

Create a shared.css as your single source of truth. Every page links to this file. When you change a token, ALL pages update.

AVOID THIS

Don't let AI use inline style="" attributes when a CSS class exists. Inline styles bypass your design system and guarantee inconsistency.

ATTENTION HOOK вҖ” "IT WORKS ON MY MACHINE"

You built a beautiful prototype. It works locally. You show it to someone вҖ” fonts wrong, layout breaks, images don't load.

The gap between "prototype" and "production" is systematic: relative paths, CDN dependencies, error states, loading states, mobile breakpoints, accessibility, SEO meta tags, and deployment config.

Why This Matters

Your pain point: "Continuous iteration due to simple CSS errors" and "Changing outputs here and there."

These errors happen because iteration is uncontrolled вҖ” you fix one thing, break another. Production workflow: test before deploy, version changes, never push untested code.

Core Concept: The 8-Point Hardening Checklist

CRITICAL (must pass before deploy):
[ ] 1. All paths are relative (../shared.css, not /Users/me/...)
[ ] 2. External dependencies have CDN fallbacks
[ ] 3. All images have alt text + relative paths
[ ] 4. Mobile responsive: tested at 375px, 768px, 1024px
[ ] 5. All interactive elements work (buttons, links, forms)

IMPORTANT (should pass):
[ ] 6. Semantic HTML: lang, heading hierarchy, aria-labels
[ ] 7. Meta tags: title, description, Open Graph
[ ] 8. Performance: no images > 500KB, no unused CSS/JS

The Iteration Safety Net

Problem: You fix a CSS bug on page 3, it breaks page 1.

Solution: Always edit shared.css tokens, never inline styles. Token changes propagate safely.

Safe WorkflowDangerous Workflow
1. Make change in shared.css1. Make change
2. Check changed page2. Push immediately
3. Spot-check 1 other page3. Discover it broke something
4. Commit with message4. Panic-fix
5. Push (auto-deploy)5. Push again, repeat
CROSS-DOMAIN вҖ” SOFTWARE CI/CD

In software engineering, nobody deploys manually. Push code, automated tests run, if tests pass, auto-deploy. Apply the same: Git + GitHub Pages + deploy workflow = push to live in 30 seconds.

BEST PRACTICE вҖ” GIT COMMIT MESSAGES
DO THIS
git commit -m "fix: sidebar nav links in level-2 page"
git commit -m "feat: add audio players to level-1 sessions"
git commit -m "style: increase body font to 20px"
AVOID THIS
git commit -m "update"
git commit -m "fix stuff"
git commit -m "asdf"

GitHub Pages Deploy Workflow

# .github/workflows/deploy.yml
name: Deploy to GitHub Pages
on:
  push:
    branches: ["main"]
  workflow_dispatch:
permissions:
  contents: read
  pages: write
  id-token: write
concurrency:
  group: "pages"
  cancel-in-progress: false
jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: '.'
      - id: deployment
        uses: actions/deploy-pages@v4

LEVEL 2 SELF-CHECK QUIZ
Q1: What are design tokens?
  • A) Free icons from the internet
  • B) Named CSS variables replacing hardcoded values
  • C) Login credentials for design tools
  • D) Placeholder images
B вҖ” Design tokens are named variables (like --slate: #4E4B66) that ensure AI uses YOUR specific values instead of inventing new ones.
Q2: AI-generated page uses 3 different grays. Root cause?
  • A) The AI model is outdated
  • B) You didn't specify color tokens in the prompt/context
  • C) Gray is hard for AI
  • D) You need a different AI
B вҖ” Without explicit color tokens, AI defaults to its own interpretation. Load your shared.css with exact values.
Q3: Purpose of the 5-point validation checklist?
  • A) To slow things down
  • B) Catch systematic errors before they compound
  • C) To impress stakeholders
  • D) Generate documentation
B вҖ” Each uncaught error in iteration 1 becomes 2-3 errors by iteration 5. The checklist prevents compounding.
Q4: Fixed CSS on Level 2 and broke Level 1. What to do differently?
  • A) Edit shared.css tokens instead of inline styles
  • B) Use a different browser
  • C) Fix Level 1 first
  • D) Avoid CSS entirely
A вҖ” Changes to shared.css tokens propagate consistently. Inline edits on one page bypass the system.
Q5: Minimum for automated deployment?
  • A) A paid hosting service
  • B) GitHub repo + GitHub Pages deploy workflow (free)
  • C) A DevOps team
  • D) Docker and Kubernetes
B вҖ” GitHub Pages + a simple deploy YAML = automated deployments for free. Push to live in 30 seconds.
