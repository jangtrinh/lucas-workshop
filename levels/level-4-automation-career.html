4.1 AI AGENTS FOR REPETITIVE WORK

SESSION 4.1
TRANSCRIPT

Session 4.1. AI Agents cho công việc lặp đi lặp lại. Stop doing what AI can do for you.

NGHE — ATTENTION HOOK
TRANSCRIPT

Một PM theo dõi công việc hàng tuần của mình và phát hiện ra bốn mươi phần trăm là công việc lặp đi lặp lại: status updates, meeting notes, ticket triage, report formatting. Đó là mười sáu tiếng mỗi tuần mà máy có thể làm. PM đó xây ba agents đơn giản: một cho meeting summaries, một cho ticket categorization, một cho weekly reports. Thời gian tiết kiệm được: mười hai tiếng mỗi tuần. Tương đương hai mươi bốn nghìn đô la tiết kiệm mỗi năm.

ATTENTION HOOK — 12 HOURS RECLAIMED PER WEEK

A PM tracked their weekly tasks: 40% was repetitive — status updates, meeting notes, ticket triage, report formatting. That's 16 hours/week of work a machine could do.

The PM built 3 simple agents: one for meeting summaries, one for ticket categorization, one for weekly reports. Time reclaimed: 12 hours/week. Salary equivalent saved: $24,000/year.

Why This Matters

Lucas's pain — PM time eaten by repetitive tasks. Every hour spent on copy-paste work is an hour NOT spent on strategy, user research, or product decisions. The ceiling isn't skill — it's hours.

Core Concept: The Agent Pattern

NGHE — KHÁI NIỆM CHÍNH
TRANSCRIPT

Agent Pattern có bốn bước. Trigger: sự kiện xảy ra — email mới, ticket mới, hoặc theo lịch. Decide: AI đọc context và quyết định phải làm gì. Execute: Function Calling kích hoạt các hành động thực tế trong hệ thống thật. Report: Structured Output JSON được lưu vào knowledge base. Ba khả năng Gemini quan trọng: Function Calling cho phép AI gọi các APIs của bạn, Code Execution chạy Python trong sandbox miễn phí, và Structured Output đảm bảo format JSON nhất quán.

│ Event    │─────▶│ AI reads │──────▶│ Function │─────▶│ Structured│
│ happens  │      │ context  │       │ Calling  │      │ Output   │
│          │      │ + decides│       │ runs     │      │ (JSON)   │
│ New email│      │ what to  │       │ real     │      │ saved to │
│ New ticket│     │ do       │       │ actions  │      │ your KB  │
│ Schedule │      │          │       │          │      │          │
└─────────┘      └──────────┘       └──────────┘      └──────────┘

Key Gemini capabilities:
  Function Calling   → Model outputs structured calls to YOUR APIs.
                       You define the functions, AI decides when/how to call them.
  Code Execution     → Python sandbox for data processing, calculations, chart generation.
                       FREE, no setup required.
  Structured Output  → JSON schema enforcement — agent always returns predictable format.

CROSS-DOMAIN — MANUFACTURING (FORD ASSEMBLY LINE)

Henry Ford didn't speed up hand-building cars. He created an assembly line where each station does ONE task. AI agents work the same way — each agent has one job, one trigger, one output format. Chain them for complex workflows.

EXERCISE — SPECIFY YOUR FIRST AGENT

Identify 3 repetitive tasks, then spec the simplest one

For each of your top 3 repetitive tasks, define:

  1. Trigger — what event starts this task? (new email, new Jira ticket, scheduled time)
  2. Decision — what does the AI need to figure out? (classify, summarize, route)

  4. Output schema — what JSON structure does it return?

Pick the simplest task and write its full agent specification below.

Before automating, score each task. If Learning Value > 3, add a human checkpoint — don't fully automate.

Session 4.2. Career Pipeline. Từ job posting đến tailored application trong ba mươi phút.

NGHE — ATTENTION HOOK
TRANSCRIPT

Lucas nộp đơn vào năm công ty mỗi tuần. Mỗi đơn cần tailored CV, cover letter, và portfolio selection. Cách thủ công: ba tiếng mỗi đơn nhân năm bằng mười lăm tiếng mỗi tuần chỉ để nộp đơn. Với career pipeline: ba mươi phút mỗi đơn nhân năm bằng hai tiếng rưỡi mỗi tuần. Mười hai tiếng rưỡi được giải phóng mỗi tuần.

ATTENTION HOOK — 15 HOURS/WEEK TO 2.5 HOURS

Lucas applies to 5 jobs per week. Each needs a tailored CV, cover letter, and portfolio selection. Manual approach: 3 hours × 5 = 15 hours/week on job apps alone.

With a career pipeline: 30 minutes × 5 = 2.5 hours/week. 12.5 hours reclaimed every single week.

Why This Matters

Lucas's pain point — job applications. The problem isn't writing CVs. It's that each application requires re-reading the posting, re-selecting relevant experience, re-writing to match keywords, re-formatting. A pipeline does this systematically — every time, without fatigue.

EXERCISE — BUILD YOUR CAREER KNOWLEDGE BASE

  4. Test it: paste a real job posting URL, then manually identify which of your achievements match which requirements
  5. Define output format: what should a "tailored CV section" look like? (heading structure, word count, tone)

ATTENTION HOOK — THE SWISS ARMY KNIFE PROBLEM

You've learned 12+ AI capabilities across 4 levels. Using them one at a time is like having a Swiss Army knife and only using the blade.

The orchestrator pattern chains multiple capabilities into one workflow — triggered once, produces complete output. Everything in Levels 1–4 connects here.

CROSS-DOMAIN — ORCHESTRA CONDUCTING

A conductor doesn't play every instrument. They set the tempo, cue each section at the right moment, and ensure harmony. Your orchestrator prompt does the same: it sequences capabilities, passes context between them, and ensures the final output is coherent.

Approach Comparison

ApproachCapabilities UsedCoherenceSetup
Single prompt1High (simple)0
Sequential prompts3–5 (manual)Medium (context lost between)Low
OrchestratorAll availableHigh (shared context)2–4 hours (once)
BEST PRACTICE
DO THIS

Start with a 2-step orchestrator (e.g., Research → Create). Add steps one at a time. Test each step independently before chaining. Validate step 1 output before wiring it to step 2.

AVOID THIS

Building a 6-step orchestrator on day one. Debug hell. A broken step 3 in a 6-step chain means re-running everything. Start narrow, expand after each step is validated independently.

NGHE — BÀI TẬP CAPSTONE
TRANSCRIPT

Capstone project. Thiết kế một orchestrator cho thách thức lớn nhất của bạn. Định nghĩa trigger — cái gì bắt đầu workflow. Liệt kê các steps với capabilities nào theo thứ tự nào. Định nghĩa context flow — step 1 pass gì cho step 2. Định nghĩa final output format cụ thể. Và xác định một step có khả năng thất bại nhất — fallback của bạn là gì.

CAPSTONE PROJECT — DESIGN YOUR ORCHESTRATOR

From Solo PM to AI Team — design the workflow that changes your work

Design an orchestrator for YOUR biggest challenge:

  1. Define the trigger — what single event starts the entire workflow?
  2. List the steps — which capabilities, in what order? (minimum 3 from different levels)
  3. Define context flow — what does step 1 output, and what does step 2 receive?
  4. Define the final output format — not "a report" but "a Markdown file with X sections containing Y"
  5. Identify the riskiest step — most likely to fail — and define your fallback

LEVEL 4 SELF-CHECK QUIZ
Q1: An AI agent differs from a chatbot because:
  • A) It uses a better model
  • B) It can trigger real external actions via function calling
  • C) It responds faster
  • D) It costs less
B — A chatbot converses. An agent acts. Function calling lets the model output structured API calls that run real code against real systems (Slack, Jira, email). The distinction is not model quality or speed — it's whether something happens in the world as a result.
Q2: Best first agent to build?
  • A) The most impressive one
  • B) The most complex one
  • C) The most repetitive + simplest task
  • D) The one your boss asked for
C — High frequency means you see the value (and catch bugs) immediately. Low complexity means debug cycles are fast. Build the boring agent first. The impressive ones come after you've validated the pattern on something simple.
Q3: Career Pipeline requires which prerequisite?
  • A) A paid AI subscription
  • B) A structured knowledge base of your achievements
  • C) Professional CV template
  • D) 10+ years of experience
B — The pipeline is assembly, not invention. It matches job requirements against YOUR achievements to produce tailored output. Without a knowledge base, it hallucinates achievements you don't have, which is worse than a generic CV.
Q4: Context Caching in the Orchestrator saves money because:
  • A) It uses a cheaper model
  • B) Shared context across steps costs 90% less than re-sending
  • C) It compresses the output
  • D) It skips unnecessary steps
B — Context Caching stores your large context (product docs, KB, design tokens) server-side with a TTL. Each orchestrator step reads from cache at ~10% of normal input token cost instead of re-uploading the full context for every step.
Q5: Most important lesson from this workshop?
  • A) Use more AI tools
  • B) Write better prompts
  • C) Build SYSTEMS that produce consistent output, not one-off prompts
  • D) Automate everything immediately
C — Tools and prompts are tactics. Systems are strategy. One-off prompts produce one-off results. Systems (pipelines, agents, orchestrators) produce compounding results — each cycle faster and better than the last because the infrastructure already exists.
