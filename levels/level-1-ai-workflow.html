1.1 THE SYSTEM, NOT THE PROMPT

SESSION AUDIO
TRANSCRIPT

Chào mừng bạn đến với session 1.1: Hệ thống, không phải câu lệnh. Việc nhập câu lệnh lặp đi lặp lại khiến bạn lãng phí hàng giờ mỗi tuần. Nền tảng của một AI workflow hiệu quả là xây dựng một lớp ngữ cảnh cố định, giúp AI luôn ghi nhớ thông tin sản phẩm và thiết kế của bạn.

ATTENTION HOOK — THE $2,400 PROMPT PROBLEM

A PM at a Series B startup spent 3 hours/day re-explaining their product to ChatGPT across 47 separate conversations. That's 15 hours/week x $80/hr = $1,200/week of PM salary wasted on context re-entry.

The fix wasn't a better prompt. It was a system: one persistent workspace with product context, design tokens, and personas loaded once. Re-prompting time dropped to 12 minutes/day.

Question for you: How many times this week did you re-explain the same thing to an AI?

Why This Matters

Every pain point traces back to one root cause: context loss.

Pain PointRoot Cause
"Changing outputs after each iteration"AI doesn't remember your decisions
"Generic outputs for design/specs"AI doesn't know your product context
"AI slop — undervalued, obvious content"AI doesn't know your audience's level
"Inconsistent design output"AI doesn't have your design system
"Not truly tailor-made"AI doesn't have your specific constraints
KEY INSIGHT

One system fixes all five.

Core Concept: The 3-Layer AI Workflow

┌─────────────────────────────────────┐
│  Layer 3: TASK PROMPT               │  ← What you ask (changes each time)
│  "Write a PRD for feature X"        │
├─────────────────────────────────────┤
│  Layer 2: SKILL INSTRUCTIONS        │  ← How to do it (reusable templates)
│  Design rules, writing style,       │
│  output format, quality criteria    │
├─────────────────────────────────────┤
│  Layer 1: CONTEXT FOUNDATION        │  ← Who you are (load once, use always)
│  Product brief, target audience,    │
│  design tokens, competitor data,    │
│  past decisions, brand voice        │
└─────────────────────────────────────┘

Most people only build Layer 3 (the prompt). That's why outputs are generic. Consistency comes from Layers 1 and 2.

CROSS-DOMAIN — GAME DESIGN SAVE STATES

Video games solved this decades ago. Imagine if every time you opened a game, you started from Level 1 with no inventory. That's what ad-hoc prompting is — no save state. The 3-Layer system is your save file: Layer 1 = your character (product context), Layer 2 = your abilities (skill templates), Layer 3 = the current quest (task prompt).

Comparison: Approaches to AI Workflow

ApproachSetupPer-TaskQualityConsistency
Ad-hoc prompting0 min30-60 min3-7/10Low
Prompt library2 hrs15-20 min5-7/10Medium
3-Layer System4 hrs (once)5-10 min7-9/10High
BEST PRACTICE
DO THIS

Build your Context Foundation (Layer 1) BEFORE writing any prompts. Spend 2 hours documenting: product (what, who, problem), design (tokens, typography, components), voice (formal/casual/technical), constraints (budget, timeline, tech stack).

AVOID THIS

Don't try to put everything in one mega-prompt. Layer 1 should be a separate document that gets loaded into your AI workspace — not copy-pasted into every conversation.

1.2 RESEARCH THAT COMPOUNDS

SESSION AUDIO
TRANSCRIPT

Session 1.2: Research That Compounds. Thay vì vứt bỏ lịch sử chat sau mỗi lần nghiên cứu, hãy lưu trữ dữ liệu dưới định dạng có cấu trúc như CSV. Khi AI có thể truy vấn một cơ sở kiến thức ngày càng lớn, chất lượng đầu ra của bạn sẽ tăng dần theo thời gian.

ATTENTION HOOK — THE GOLDFISH MEMORY PROBLEM

You run a great research session with AI: 45 minutes, 12 sources, deep competitor analysis. Then you close the tab. Next week, you need the same data — but you start from zero. The research doesn't compound. It evaporates.

Meanwhile, the best PMs build knowledge bases — structured data that gets richer over time. After 3 months, they have an unfair advantage: AI outputs grounded in 200+ data points instead of 5.

Bridge

In Session 1.1, you built a Context Foundation — the "who" and "what." Now we add the "market" layer: competitors, trends, benchmarks, and validated data points.

Why This Matters

Your pain point: "Generic outputs for design, specs that cannot really compete in market."

Generic outputs happen when AI has no market context. It defaults to textbook answers because it doesn't know YOUR specific competitive landscape.

Key insight: The format matters more than the volume. 20 well-structured data points beat 200 unstructured paragraphs.

CROSS-DOMAIN — JOURNALISM'S INVERTED PYRAMID

Journalists structure information with the most important facts first. Apply this to your knowledge base: P0 = hard numbers, P1 = positioning/strengths, P2 = trends/opinions.

BEST PRACTICE
DO THIS

After every research session, extract findings into structured format with columns: source, category, insight, confidence, date.

AVOID THIS

Don't dump raw web pages into your knowledge base. AI can't distinguish signal from noise in unstructured data. Always: clean > tag > score before storing.

EXERCISE

Run a 20-Minute Research Sprint

Pick one of your projects and run a focused research sprint:

  1. Define the question (1 min): What specific thing do you need to know?
  2. Search (10 min): Use web search + AI to find answers
  3. Structure (5 min): Extract into CSV format
  4. Store (2 min): Save the CSV in your project directory
  5. Test (2 min): Load the CSV into AI and ask a question that requires the data

1.3 BRIEFING AI LIKE A CEO

SESSION AUDIO
TRANSCRIPT

Session 1.3: Khởi tạo brief như một CEO. Thay vì ra lệnh chung chung, hãy sử dụng framework BRIEF: Background, Requirements, Intent, Examples và Format. Một brief rành mạch sẽ triệt tiêu vòng lặp sửa lỗi, giúp AI đưa ra kết quả chất lượng ngay ở lần chạy đầu tiên.

ATTENTION HOOK — THE CEO TEST

When a CEO briefs their team, they don't say: "Make something good." They say: Target — reduce churn by 15% in Q2. Constraint — no new hires. Reference — how Spotify does onboarding. Anti-pattern — don't copy competitors' pricing page. Deliverable — 3-slide deck by Friday.

That brief produces predictable output because it defines what good looks like upfront. Your AI prompts should work the same way.

Bridge

In Session 1.1, you built your Context Foundation (Layer 1). In Session 1.2, you built your Knowledge Base. Now we build Layer 2: Skill Instructions — reusable templates that tell AI HOW to execute.

Why This Matters

Your pain point: "Continuous iteration due to simple CSS errors" and "Changing outputs here and there after each iteration."

These aren't AI problems. They're briefing problems. The AI doesn't know your CSS conventions, your layout rules, or what "done" looks like. Fix the brief, fix the output.

Core Concept: The BRIEF Framework

B — Background:   Load your Context Foundation (Layer 1)
R — Requirements:  What MUST the output include? What MUST it avoid?
I — Intent:        WHY do you need this? What decision does it support?
E — Examples:      Show 1 good example and 1 bad example
F — Format:        Exact output structure (HTML? Markdown? CSV? JSON?)

Bad Brief vs. Good Brief

BAD BRIEF (PRODUCES GENERIC OUTPUT)

"Create a landing page for my product"

GOOD BRIEF (PRODUCES PRODUCTION OUTPUT)
## Background
[Load CONTEXT.md — product, audience, design constraints]

## Requirements
MUST include:
- Hero section with headline + subheadline + CTA button
- Social proof section (3 testimonials)
- Feature grid (3 columns, icon + title + description)
- FAQ section (5 questions)

MUST avoid:
- Gradients, shadows, border-radius (monochrome design system)
- Generic stock photo language ("team of professionals")
- More than 2 font families

## Intent
This landing page collects email signups for beta launch.
Success = 100 signups in first week. Optimize for conversion.

## Examples
GOOD: [link to a landing page you admire]
BAD: [link to what you DON'T want]

## Format
- Single HTML file with inline CSS
- Mobile-responsive (min-width: 320px)
- Semantic HTML5 elements
- Copy button on any code blocks

Brief Quality vs. Output Quality

Brief QualityIterationsOutputTime
"Make a landing page"8-124/103+ hours
Requirements only4-66/101.5 hours
Full BRIEF framework1-28/1030 min
BEST PRACTICE
DO THIS

Create a briefs/ directory. Save every brief as a .md file. Next time you need something similar, copy, modify, run. Your second brief for the same task type should take 5 minutes.

AVOID THIS

Don't over-specify creative tasks. The BRIEF framework is for predictable tasks (landing pages, PRDs, reports), not brainstorming.

EXERCISE

Write a Production Brief

Pick your hardest pain point and write a full BRIEF:

  • Option A (Teaching): Brief AI to create one workshop session page in HTML
  • Option B (Product): Brief AI to write a competitive analysis
  • Option C (Job): Brief AI to generate a tailored CV for a specific posting

Use the BRIEF framework. Include at least 3 MUST-include, 2 MUST-avoid, 1 good example, exact output format.

SUCCESS CRITERIA
APPLY CHALLENGE

Run your brief through AI. Score the output 1-10 on: Accuracy, Completeness, Consistency, Originality. If average < 7, identify which BRIEF section was too vague and strengthen it.

LEVEL 1 SELF-CHECK QUIZ
Q1: Why do ad-hoc prompts produce inconsistent output?
  • A) AI models are random
  • B) Each conversation starts without product context (cold start problem)
  • C) You need a more expensive AI model
  • D) Prompts need to be longer
B — The cold start problem means AI has no memory of your product, design system, or past decisions. A persistent Context Foundation solves this.
Q2: What's the difference between a prompt library and a 3-Layer System?
  • A) Nothing, they're the same
  • B) A prompt library is Layer 3 only; 3-Layer includes Context and Skills
  • C) A 3-Layer System requires paid tools
  • D) A prompt library is more advanced
B — A prompt library gives reusable templates (Layer 3), but without Context (Layer 1) and Skill Instructions (Layer 2), outputs remain generic.
Q3: What format should your research knowledge base use?
  • A) Google Docs
  • B) Screenshots
  • C) Structured CSV/Markdown with tagged fields
  • D) Bookmarked URLs
C — Structured data with columns (source, category, insight, confidence, date) allows AI to query and reason over your research.
Q4: In the BRIEF framework, what does "I" stand for?
  • A) "Implementation" — programming language
  • B) "Intent" — WHY you need this
  • C) "Input" — what data to use
  • D) "Iteration" — how many versions
B — Intent helps AI understand the decision context. A landing page for "beta signups" will be designed differently than one for "impressing investors."
Q5: Output scores 5/10 on consistency. Which layer to fix first?
  • A) Layer 3 (rewrite the prompt)
  • B) Layer 2 (add style/format instructions)
  • C) Layer 1 (strengthen Context Foundation)
  • D) Switch to a different AI model
C — Consistency issues almost always trace back to missing context. Strengthen Layer 1 (design tokens, brand voice, anti-patterns) before touching the prompt.
